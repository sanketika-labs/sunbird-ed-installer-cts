HOW TO USE THE CONVERTED INFRASTRUCTURE
================================================================================

STEP 1: Install OpenTofu
--------------------------
curl -fsSL https://get.opentofu.org/install-opentofu.sh -o install-opentofu.sh
chmod +x install-opentofu.sh
sudo ./install-opentofu.sh --install-method standalone
rm install-opentofu.sh

STEP 2: Verify Installation
----------------------------
cd opentofu/aws
./verify-opentofu-setup.sh

STEP 3: Use Your Existing Infrastructure
-----------------------------------------
# Your existing Terraform state works immediately!
cd opentofu/aws/your-environment
source tf.sh
terragrunt init      # Terragrunt auto-detects OpenTofu
terragrunt plan      # Should show no changes to existing resources
terragrunt apply     # Works exactly like before

STEP 4: New Installation (if starting fresh)
---------------------------------------------
cd opentofu/aws
cp -r template my-environment
cd my-environment
# Edit global-values.yaml with your settings
./install.sh

IMPORTANT NOTES
================================================================================

1. TERRAGRUNT AUTO-DETECTION
   Terragrunt (0.48.0+) automatically detects and uses OpenTofu when installed.
   The command "terragrunt" stays the same - it's smart enough to find "tofu".

2. COMMAND EQUIVALENCE
   terraform init    →  tofu init
   terraform plan    →  tofu plan
   terraform apply   →  tofu apply
   terraform destroy →  tofu destroy
   
   With Terragrunt, just use: terragrunt [command]

3. STATE COMPATIBILITY
   - OpenTofu reads Terraform state files ✅
   - Terraform 1.6+ reads OpenTofu state files ✅
   - No migration or conversion needed ✅

4. PROVIDERS
   Both tools use: registry.terraform.io/hashicorp/aws
   All providers work identically with both tools.

5. IF SOMETHING GOES WRONG
   You can always go back to Terraform:
   export TERRAGRUNT_TFPATH=terraform
   
   All configuration files work with both tools!
